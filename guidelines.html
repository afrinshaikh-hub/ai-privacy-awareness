Safe & Ethical AI Use

Practical rules and real examples to reduce privacy and security risk when using AI tools. The goal is to keep the benefits of AI while avoiding common mistakes that expose personal data, confidential work, or sensitive information.

What you should NOT paste into AI tools (red lines)

A simple rule: if it would be damaging if leaked, copied, or shared outside your control, do not paste it into an AI tool.

  • Passwords, API keys, auth tokens: GitHub tokens, AWS keys, school/work logins, “.env” files.
  • Financial identifiers: bank account numbers, card numbers, tax documents, payment screenshots.
  • Government IDs: SSN, passport numbers, driver’s license images, visa paperwork.
  • Medical / highly personal data: diagnosis details, prescriptions, therapy notes, private family info.
  • Confidential school/work content: internal emails, non-public documents, private source code.
  • Anything you wouldn’t post publicly: if you’d be uncomfortable seeing it online, don’t share it.

Example: Instead of pasting “Here is my resume with my address and phone number,” paste a version with placeholders: “Here is my resume (address removed). Please improve the bullet points.”

Safer alternatives to oversharing (use these instead)

  • Use placeholders: replace names with “NAME,” companies with “COMPANY,” IDs with “ID#”.
  • Summarize instead of uploading: describe the situation and only include what’s necessary.
  • Share small excerpts: paste only the relevant paragraph instead of the entire document.
  • Remove metadata: screenshots and PDFs can contain hidden details—crop and redact first.

Example: If you want AI help debugging, don’t paste private API keys or full config files. Paste the error message + the relevant function only.

Security habits when using AI for school, work, or projects

  • Assume your prompt could be stored: treat prompts like semi-public notes.
  • Use separate accounts if needed: personal vs school/work AI usage to reduce mixing sensitive info.
  • Check privacy settings: look for history, data controls, and training preferences if available.
  • Don’t trust outputs blindly: verify security advice, code suggestions, and links.
  • Be careful with “paste your logs”: logs can include tokens, internal URLs, usernames, file paths.

Example: If AI generates a command to run in your terminal, read it first. A single command can delete files, leak credentials, or expose system details if misused.

Watch out for AI-enabled scams (phishing, deepfakes, and prompt tricks)

AI can make scams look more realistic. Attackers can generate persuasive emails, fake voices, or social media messages that appear legitimate.

  • Phishing: AI-written messages that sound professional and personalized.
  • Deepfakes: fake audio/video that imitates someone you know or a public figure.
  • Prompt injection: malicious text that tries to manipulate an AI system into revealing data or ignoring rules.

Example: If you receive a “professor” email asking you to click a link and “verify your login,” verify through an official channel first. AI makes these messages harder to spot.

Ethical use: how to use AI without crossing the line

  • Be honest about AI assistance: follow your course or workplace policy for AI usage.
  • Avoid plagiarism: use AI as a helper (outline, feedback, clarity), not as a replacement for your work.
  • Respect privacy of others: don’t upload classmates’ work, private conversations, or sensitive info about people.
  • Avoid harmful profiling: don’t use AI to make decisions about people based on stereotypes or sensitive traits.

Example: If you use AI to improve writing, keep your original ideas and citations, and use AI for clarity and organization—not for inventing sources or facts.

Verification checklist (especially for security and facts)

AI tools can be confidently wrong. Before trusting outputs, do quick verification.

  • For code: read it line by line, test it, and check for unsafe patterns (hardcoded secrets, risky commands).
  • For facts: cross-check with reliable sources (official docs, credible organizations, research papers).
  • For links: don’t click unknown links from AI-generated content—type trusted URLs manually.
  • For security advice: confirm against standards or trusted security resources.

Quick Do / Don’t Summary

  • Do: use placeholders, minimize data shared, verify outputs, and keep sensitive info offline.
  • Don’t: paste passwords, IDs, confidential files, private codebases, or anything you can’t afford to leak.

If you follow these habits, you can still use AI effectively while lowering privacy and security risk.
